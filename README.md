# esp32WifiConnect

# 概要
ESP32のAP機能を利用して、ノートPCからESP32のSSIDとIPを見つけて自動的に接続するアプリ。

# 背景と目的
- ESP32搭載のロボットをwifi経由で制御する際、外部ノートPCでESP32のSSIDを手動で見つけて接続する必要があり、手間がかかる。
- 外部ノートPCから簡単にESP32のAPに接続できるアプリを作成する。

# 想定ユーザ
- ノートPCユーザ（主に Windows）
- ESP32搭載ロボットを Wi-Fi 経由で操作・デバッグする開発者・運用者

## 機能の概要

ノートPC上で動作するデスクトップアプリ（または CLI ツール）が、周辺の Wi-Fi ネットワークをスキャンし、ESP32 が起動した AP（アクセスポイント）を自動検出して接続する。接続後、ESP32 の IP アドレスを取得し、接続状態をユーザーに表示する。

### 基本フロー
1. ユーザーがアプリを起動する
2. アプリが Wi-Fi インターフェースの利用可否を確認する
3. 周辺 AP をスキャンし、事前に登録しておいたESP32と判定できる SSID を抽出する
4. AP へ接続する（必要に応じてパスワードを入力または設定ファイルから読み込む）
6. 接続完了後、ESP32 の IP アドレス（通常はゲートウェイ IP）を表示する
7. 接続状態（SSID / IP / 接続中・切断）を画面に表示する

### 対象とする ESP32 側の前提
- ESP32 は **SoftAP モード** で動作している
- SSID は判別可能な命名規則に従う（例: `ESP32-` プレフィックス、または設定ファイルで指定したプレフィックス）
- AP モード時の IP は ESP32 標準の `192.168.4.1` を基本とする（変更されている場合は設定で上書き可能）

## 必須の機能

| ID | 機能 | 説明 |
|----|------|------|
| F-01 | Wi-Fi スキャン | ノートPCの Wi-Fi から周辺 AP 一覧を取得する |
| F-02 | ESP32 AP の検出 | SSID の命名規則（プレフィックス一致など）に基づき ESP32 AP を候補として抽出する |
| F-03 | AP 一覧表示 | 検出した候補 AP を SSID・信号強度（RSSI）付きで表示する |
| F-05 | Wi-Fi 接続 | AP へ接続する |
| F-06 | IP アドレス表示 | 接続後、ESP32 へアクセス可能な IP アドレスを表示する |
| F-07 | 接続状態表示 | 現在の接続 SSID・IP・接続状態を画面に表示する |
| F-08 | 切断 | ユーザー操作で ESP32 AP から切断できる |
| F-09 | 設定 | SSID プレフィックス、パスワード、ESP32 IP などを設定ファイルで変更できる |

### 設定項目（必須）

| 項目 | デフォルト値 | 説明 |
|------|-------------|------|
| `ssid_prefix` | `ESP32` | この文字列で始まる SSID を ESP32 候補とみなす |
| `ap_password` | （空） | AP パスワード。空の場合は接続時にユーザー入力 |
| `esp32_ip` | `192.168.4.1` | 接続後に表示・利用する ESP32 の IP |
| `scan_interval_sec` | `5` | 自動再スキャンの間隔（秒）。0 で手動のみ |

## 制約事項

### プラットフォーム
- **第1フェーズの対象 OS: Windows 10 / 11**
- Wi-Fi 機能を備えたノートPCが必要
- 管理者権限は原則不要とする（Windows の Wi-Fi 接続 API / `netsh` 等で一般ユーザー権限で接続可能な方法を採用する）

### ネットワーク
- ESP32 が SoftAP として起動していること
- ノートPC の Wi-Fi が有効であること
- 同一時間に接続できる ESP32 AP は **1 台** を基本とする（複数台同時接続は対象外）
- インターネット経由の接続は行わない（ローカル AP への直接接続のみ）

### セキュリティ
- パスワードを設定ファイルに平文保存する場合、ローカル利用に限定し、リポジトリへのコミットは禁止する
- `.gitignore` で設定ファイル（例: `config.local.json`）を除外する

### 非対象（第1フェーズ）
- ESP32 側ファームウェアの開発・書き込み
- ロボット制御プロトコル（HTTP / WebSocket / シリアル等）の実装
- macOS / Linux 対応
- モバイル（Android / iOS）対応
- STA モード（ESP32 が既存 Wi-Fi に参加している場合）での IP 探索

## 画面要件（GUI）

| 要素 | 内容 |
|------|------|
| スキャンボタン | 周辺 AP を再スキャン |
| AP 一覧 | SSID / 信号強度 / 選択 |
| 接続ボタン | 選択 AP へ接続 |
| 切断ボタン | 現在の接続を解除 |
| 状態表示 | 接続 SSID、ESP32 IP、接続状態 |
| 設定 | SSID プレフィックス、パスワード、ESP32 IP |

## 非機能要件

| 項目 | 目標 |
|------|------|
| スキャン完了時間 | 起動後 10 秒以内に候補 AP を表示 |
| 接続完了時間 | AP 選択後 30 秒以内に接続完了またはエラー表示 |
| 操作性 | Wi-Fi 設定画面を開かずに AP 接続まで完結 |
| エラー表示 | 接続失敗時は原因（パスワード誤り、AP 未検出、Wi-Fi 無効など）を日本語で表示 |

## エラー処理

| 状況 | アプリの動作 |
|------|-------------|
| Wi-Fi が無効 | 「Wi-Fi を有効にしてください」と表示 |
| ESP32 AP が見つからない | 「ESP32 が見つかりません。電源と AP モードを確認してください」と表示 |
| パスワード誤り | 再入力を促す |
| 接続タイムアウト | 再スキャン・再接続を案内 |
| 既に別ネットワークに接続中 | 接続前に確認ダイアログを表示 |

## 用語

| 用語 | 説明 |
|------|------|
| AP（Access Point） | ESP32 が自身で提供する Wi-Fi 基地局モード（SoftAP） |
| SSID | Wi-Fi ネットワーク名 |
| SoftAP | ESP32 が AP として動作するモード |
| RSSI | 電波強度の指標。値が大きいほど近い |
